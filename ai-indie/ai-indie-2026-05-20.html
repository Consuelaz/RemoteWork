AI独立派

我今年一行代码没写，合并了 150 个 PR——这才是 AI 时代真正的分水岭

本文由 AI 辅助整理，观点源于独立开发实战与真实 builders 的一手声音。

"2026 年至今，我没有手写过一行代码，但我每天合并几十个 PR，最高一天 150 个。"

这是 Claude Code 的创始工程师 Boris Cherny，在接受红杉资本采访时说的话。

他不是在炫耀，也不是在开玩笑。他描述的，是他目前真实的每日工作状态：手机上同时挂着几百个 AI 智能体在跑，每晚还有几千个在做深度异步任务，他本人负责的，只有方向和判断。

当你读到这句话，第一反应是什么？羡慕？恐慌？还是觉得"这跟我有什么关系"？

如果是最后一种，我想让你停下来再想想。因为这件事正在划出这个时代独立开发者之间，一条越来越深的分水岭——不在于你会不会用 AI，而在于你怎么用。

01 / 从"写代码"到"管智能体"，不是升级，是换岗

Boris 的工作流让很多人震惊，但 Anthropic CEO Dario Amodei 在最新专访中补了一刀：

"Anthropic 内部，多数工程主管已经不再亲自写代码。他们的日常工作是审核、编辑 Claude Opus 的输出。"

更夸张的是，最新的 Claude Co-work 产品——面向非技术人群的智能体应用——几乎完全由 Claude Opus 自主开发，人工投入不到一周半。产品上线后，关键运营指标是同类产品的 4 倍。

指标数据
Anthropic 企业付费采用率（2026年4月）34.4%，首超 OpenAI 的 32.3%
Claude Code 全球 GitHub 代码提交占比4%，过去一个月翻倍
Boris Cherny 2026年至今手写代码0 行
Boris 日均合并 PR几十个，最高单日 150 个

这些数字背后，有一个正在悄悄改变的事实："程序员"这个角色的核心工作，已经不再是"写代码"了。

以前，你的产出取决于你的编码速度。现在，你的产出取决于你同时能管多少智能体，以及你给它们的方向是否清晰。

对独立开发者来说，这不是一个关于工具的问题，而是一个关于角色认知的问题——你还在用 AI 帮你补全几行代码，还是已经在用 AI 替你跑整条生产线？

02 / Codex 上手机，真正的影响不是"随时写代码"

5 月 14 日，OpenAI 正式把 Codex 推进 ChatGPT 移动端，iOS/Android 全面开放预览。发布当天，Sam Altman 同步宣布：任何切换到 Codex 的公司，可获得两个月免费使用——时间点正好是 Anthropic Claude Code 超越 OpenAI 的那一周。

Codex 手机端的真正价值，不是"让你在地铁上写代码"。它的核心，是把控制面板装进你口袋里，让你随时随地完成最关键的一件事：给 AI 定方向，审 AI 的结果。

☕ 场景一：咖啡店里审 diff

早上派下几个任务，到点检查进度，diff 一目了然。某段改动方向不对，在手机上敲两句回复，智能体立即调整。一杯咖啡，三个 PR 的进展全部消化。

🚇 场景二：通勤路上改方向

智能体运行到一半卡住，主动向你推消息。地铁里瞄一眼，回复"用 X 库的 Y 方法"，它拿到答案继续跑。30 分钟通勤，消化两三个关键决策点。

🍱 场景三：一顿饭变一个原型

吃饭时冒出产品想法，直接掏手机派给 Agent "先做个原型"，回到工位已有可看的提交记录。从想法到代码，缩短到一顿饭。

对独立开发者来说，工作方式的瓶颈从"编码速度"彻底转移到了判断力密度——你一天能做多少次高质量决策，决定了产出上限。更重要的是，这把资源杠杆拉到了极致：一个人监督四五条线同时推进，每条线都有独立的 Agent 在跑。这是独立开发者第一次拥有真正的规模化能力。

03 / 反直觉：越不会写代码，可能越快适应这个时代

Boris Cherny 透露了一个细节：Claude Code 团队里，工程经理、产品经理、设计师、数据科学家、财务、用户研究员，全员都在用 Claude Code 写代码。等等——财务？用户研究员？

是的。在 Claude Code 团队，"写代码"这件事已经不再是程序员的专利。

这里有一个反直觉的推论：那些从一开始就没把"写代码"当作核心优势的人，反而没有什么需要放弃的——他们可以更干净地接受这套新范式，因为他们的自我认知从来不建立在"手写代码"这件事上。

"软件成本将持续下降，甚至趋于免费。但真正稀缺的，是知道要构建什么、为谁构建、为什么构建。"
—— Dario Amodei，Anthropic CEO

换句话说，代码在贬值，但需求判断力在升值。对独立开发者而言，这是一个实实在在的好消息。你和大公司之间最大的不对称优势，从来不是代码质量，而是你离你的用户更近。你更快知道"他们在为什么痛苦"，更快能测试"这个东西有没有人要"。这个能力，AI 暂时还替不了你。

04 / 这几天 builders 圈值得关注的事

Claude Code v2.1 + Agent View 研究预览版

本周上线的 Agent View，核心是在一个终端界面内统一管理所有并行 AI 会话。价值点不在于"界面好看了"，而是认知负担降低了——当你同时跑着 5-10 个 Agent，最大的心理障碍是"我不知道每个在干什么"。这是并行智能体工作流从"炫技"变成"日常"的最后一公里。值得现在就开始探索。

Anthropic 企业付费采用率首超 OpenAI

来自 Ramp AI 指数（覆盖美国 5 万余家企业数据）：2026 年 4 月，Anthropic 采用率 34.4% vs OpenAI 32.3%，增长完全来自 Claude Code。如果你还没认真测试多 Agent 工作流，现在是好时机——企业侧的需求会带来更多协作机会和工单。

Codex 手机端进入 ChatGPT App

400 万周活用户的基础盘，加上移动端入口，标志着 AI 编程 Agent 正式从"桌面工具"变成"随身工具"。这两家的竞争，本质是在争夺独立开发者的时间和注意力。你的选择，会影响哪家生态更强。

05 / 如果你今天只做一件事

ACTION 01

在下一个项目里，把"一次跑 3 个并行 Agent"当作起点去试。找一个在做的项目，拆出 3 个独立子任务，分别派给 Claude Code。哪怕其中一个跑坏了，你也会学到哪类任务 AI 能自主跑好，哪类需要你持续干预。这个认知，比任何教程都值钱。

ACTION 02

把"给 AI 下指令"当成你现在最重要的核心技能去练。Boris 今年没写一行代码，但他写了无数的任务说明、方向校正、决策反馈。你的技术能力，现在最好的出口，是精准描述问题、清晰定义边界、快速判断对错——而不是亲手实现。

ACTION 03

如果你是非技术背景，现在反而是最好的入场时机。别再觉得"我不会写代码，AI 时代跟我没关系"。Anthropic 的财务分析师在用 Claude Code 做数据脚本，用户研究员在用它处理访谈转录。你离"能用 AI 构建东西"，可能比你以为的近得多。

最后一个反转留给你：

大家都在担心 AI 会不会替代程序员。但 Boris 的故事告诉你，第一个被"替代"的，其实是 Boris 自己——他主动把自己从"写代码的人"变成了"管理 AI 写代码的人"。

这不是一种损失。这是一种杠杆。

AI 能帮你写代码，但它不知道你的用户在痛什么。这才是你的护城河。
