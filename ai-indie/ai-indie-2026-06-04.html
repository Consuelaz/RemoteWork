微软宣布 Agent 取代 App——并且把 85% 的钱留给了你

AI独立派 · 2026-06-04

"Agent 不是下一个 App。Agent 是新的应用原语。"

这是微软 CEO Satya Nadella 在 6 月 2 日 Build 2026 开发者大会上的原话。他说这句话的时候，台下坐着 2,500 名开发者，旧金山 Fort Mason 的海风从窗缝里灌进来。此前一天，Anthropic 刚完成 $650 亿 Series H 融资，估值冲到 $9,650 亿——超越 OpenAI，成为全球最有价值的私人 AI 公司。

两件事放在一起看，你得出的结论不应该只是"AI 行业真热闹"。结论是：应用开发的范式正在从"写 App"变成"造 Agent"，而平台公司已经开始为这个新范式铺路了——铺路的时候，他们选择先把钱分给你。

01 / Agent 进操作系统，不是功能更新，是物种切换

Build 2026 最大的发布不是某个模型，而是一个系统级变化：Windows Agent Runtime。

简单说，微软把 Agent 的运行时直接嵌进了操作系统 Shell。这意味着 Agent 不再是你"打开一个 App 然后调用的 AI 功能"——它变成了操作系统原生的东西，像进程管理器、像文件系统一样，是系统呼吸的一部分。

配合这个 Runtime，微软还扔出了 Windows Agent Store：开发者上传 Agent 技能，微软抽 15%，你拿 85%。做个对比：Apple App Store 是 70/30，Google Play 是 70/30。微软直接把开发者的分成从 70% 拉到 85%。

这不是慷慨。这是焦虑。

Nadella 在 keynote 里反复提一个词："Agent-first"。他的逻辑很直白：2025 年大家还在争论 Agent 该用什么标准和框架，2026 年的问题是"怎么让 Agent 真正跑起来"。而要跑起来，微软需要一个关键要素——开发者。

Windows 有 15 亿用户，但 Agent 生态才刚刚开始。微软知道，如果没有足够多的开发者在上面造 Agent 技能，Windows Agent Runtime 就是一个空壳。所以他们掏钱。

这对独立开发者意味着什么？意味着你现在面对的不是"要不要学 Agent 开发"的问题，而是"你要不要抢在别人前面占一个 Agent Store 的坑位"的问题。

02 / 独立开发者，你不需要造引擎，你只需要造一辆好车

过去两周，AI 行业密集扔出了三块拼图：

拼图一：基础设施层正在被巨头包揽。 Anthropic 估值 $9,650 亿，SpaceX 以每月 $12.5 亿的价格给 Anthropic 提供超级算力，微软自研 MAI 七款模型全部从零训练、零蒸馏。模型和算力的竞争已经变成了万亿美元级别的军备竞赛。独立开发者不需要掺和这一层。

拼图二：工具层正在变得可靠。 5 月 28 日，Claude Code 发布了史上最大规模升级——核心是"自愈"功能。Agent 现在可以在不重启会话的情况下自动绕过致命异常。过去遇到错误只能重启、丢失上下文的日子结束了。这意味着 Agent 从"玩具"变成了"可以在生产环境连续运转的东西"。与之呼应，Copilot Studio 的计算机使用代理（CUA）也在 5 月 26 日进入 GA，Token 消耗降低约 50%。

拼图三：分发层正在向开发者倾斜。 Windows Agent Store 的 85% 分成只是开始。Azure Foundry 上，模型目录同时开放 Anthropic、OpenAI、DeepSeek 等多家模型，开发者可以自由选择和调整权重。Frontier Tuning 让企业用自身工作数据定制模型，成本降低约 10 倍。

把这三块拼图拼在一起，画面很清楚：巨头们在造引擎、造赛道、修收费站。独立开发者的活，是造车——造那些真正跑在具体场景里的 Agent 应用。

你不需要和 Anthropic 比谁的模型更强，不需要和 SpaceX 比谁的算力更大。你需要的是比大公司更快地发现一个垂直场景，然后用这些现成的引擎和赛道，跑出第一辆车。

03 / 那个最反直觉的事：合规不是枷锁，是你的护城河

EU AI Act 大部分规则将于 2026 年 8 月 2 日 正式生效。这意味着两个月后，所有服务欧洲用户的 AI 产品，都需要满足透明度义务、高风险系统合规义务、AI 生成内容标注等要求。

乍一看，这又是一件"巨头才有资源应对"的事。但 Kersai 6 月 1 日的报告里有一句话点到了本质：

"最简单的 AI 采用路径不是'购买全新的 AI 堆栈'，而是'让我们已经在用的软件变得更智能'。"

这句话反过来读，就变成了独立开发者的机会：大公司有合规负担，因为他们要改造的是整个现有的 IT 系统。而你作为一个独立开发者，从零开始构建一个合规的 Agent 应用，反而更轻。

微软在 Build 上发布了两个开源工具：ACS（Agent 控制规范）和 ASSERT（开源测试框架）。ACS 定义了 Agent 能做什么、不能做什么、什么时候需要人类批准、需要记录什么证据。ASSERT 把自然语言描述的行为预期转化成结构化评分测试。

合规工具也正在被开源和标准化。过去合规是大公司的专利（因为他们请得起律师），现在合规是所有人的标配。而当一个标配出现时，先掌握它的人有先发优势。

04 / 这几天 builders 圈值得关注的事

Claude Code 自愈功能 — 5 月 28 日升级解决六大痛点：全屏渲染器消除闪烁、AI 思考过程流式输出、报错附带故障背景、长项目上下文压缩优化、MCP 协议底层稳定性提升、以及核心的"自愈"——自动绕过致命异常而不丢失会话。核心价值点不是"AI 变聪明了"，而是AI 编程工具开始具备生产环境所需的可靠性。对独立开发者来说，这意味着你可以把更多项目交给 Agent 托管，而不是每次报错都要手工重启。

微软 MAI 模型家族开放权重 — 微软自研的七款 MAI 模型（从零训练、零蒸馏），包括旗舰推理模型 MAI-Thinking-1 和轻量级编码模型 MAI-Code-1-Flash（50 亿参数），将通过 Open Router、Fireworks、Baseten 分发，开发者首次可以自行调整模型权重。核心价值点是模型选择不再绑定单一供应商——你可以用 MAI 做推理、Claude 做编码、DeepSeek 做低成本任务，自由组合的成本和灵活性优势开始显现。

Copilot Studio CUA 正式 GA — 计算机使用代理（Computer-Using Agent）正式可用，Token 消耗降低约 50%。这意味着 Agent 可以像人类一样通过视觉导航网页和桌面应用 UI。对独立开发者的直接价值：你可以让 Agent 帮你做竞品调研、自动化测试、甚至运营社交媒体——这些以前需要打开十几个标签页手动操作的重复性工作，现在可以交给一个 Agent 串行执行。

05 / 如果你今天只做一件事

第一，去了解 Windows Agent Store 的入驻规则。 不是因为你一定要用 Windows，而是因为 85% 的分成比例清楚地告诉你一件事：平台在抢开发者。当平台抢开发者的时候，早期入驻者的红利最大。就像 2008 年第一批 iOS 开发者，2016 年第一批小程序开发者——现在轮到 Agent 技能开发者了。

第二，重新审视你的产品形态。 如果你的产品还是一个"用户打开网页/App → 点击按钮 → 获得结果"的流程，问自己一个问题：它能不能变成一个"用户说一句话 → Agent 自动执行 → 推送结果"的流程？如果答案是"能"，你现在应该做的事不是加功能，而是把核心能力封装成一个 Agent 技能接口。

第三，开始用 Claude Code 或 Copilot 代理你的日常开发。 不是"偶尔让 AI 帮你写段代码"，而是真正让 Agent 托管一个完整的开发任务——从理解需求、修改代码、运行测试到提交 PR。只有你自己先成为 Agent 的使用者，你才能理解用户在使用你造的 Agent 时会遇到什么问题。

最后说一个"意外的利好"。

Nadella 在 Build 上宣布 Windows Agent Store 的时候，台下有人问：这不就是在学 Apple 的 App Store 吗？他的回答很有意思："不是。App Store 卖的是应用，Agent Store 卖的是能力。应用是你打开它才能用的东西，能力是在你需要的时候自己跑出来的东西。"

这句话的潜台词是：Agent Store 的市场规模，可能比 App Store 更大。因为 App 受限于用户主动打开的频率，而 Agent 不受限——它能 24 小时跑。

App Store 生态养活了全球超过 3,000 万开发者。Agent Store 刚刚开张，你是第一批还是第 N 批，决定权在你手上。
