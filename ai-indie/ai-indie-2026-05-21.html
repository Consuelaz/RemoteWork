AI 独立派

谷歌用 93 个 Agent 造了个操作系统——这对你意味着什么？

2026年5月21日  ·  约 2300 字  ·  独立开发者视角

本文由 AI 辅助整理，观点源于独立开发实战与真实 builders 的一手声音。

"93 个 subagent、12 小时、1.5 万次模型请求、26 亿 token，最终花了不到 1000 美元，从零搭出了一个可运行的操作系统。"

这是谷歌在 I/O 2026 现场演示 Antigravity 2.0 时，扔出来的一组数据。

坐在台下的工程师们沉默了几秒。然后，全场开始鼓掌。

这种鼓掌背后有一丝说不清楚的东西——是兴奋，还是隐约的恐惧？

对独立开发者来说，这组数字尤其刺激。以前，你一个人做一个项目，靠的是"人月神话"里那个不可能等式：一个人顶十个人用。但现在，谷歌用 93 个 Agent 造了个操作系统，还不到一千块钱。

本文核心问题只有一个：在大模型厂商用 Agent 军团互相竞争的时代，独立开发者还有位置吗？

01 / Agent 大战已经开始，战场是「连接器」，不是模型

Google I/O 2026 最值得独立开发者看清楚的，不是 Gemini 3.5 Flash 有多快（虽然快 4 倍的确诱人），而是谷歌这次真正在押注什么。

这次大会的核心叙事是 Antigravity 2.0——一个多 Agent 编排平台，分三层：桌面 App、命令行 CLI、可编程 SDK。谷歌把它定位成直接对标 Claude Code 和 OpenAI Codex 的开发者工具。

同一周，另一边，Anthropic 做了一件更耐人寻味的事：花超过 3 亿美元收购了 Stainless。

你可能没听过 Stainless。但你用过 Anthropic、OpenAI、Google 的官方 SDK——那些 TypeScript、Python、Go 版本的接口库——有一半是 Stainless 自动生成的。

Stainless 的核心能力是：给一份 API 规范，它能自动生成多语言的、符合各语言原生习惯的 SDK 和 MCP server。

Anthropic 收购它，表面上是「自建基础设施」，实质上是在宣告：AI 竞争的主战场，已经从「谁的模型更聪明」转移到了「谁的 Agent 能连上更多东西」。

对独立开发者来说，这是一个地壳运动式的信号：接下来真正值钱的，不是你会用哪个大模型，而是你能把 Agent 接进哪些「别人接不了」的系统。

02 / Gemini 3.5 Flash 降价 40%，你的项目边际成本正在崩塌

先说一个让人高兴的数字：Gemini 3.5 Flash 的价格，比上一代 Gemini 3.1 Pro 便宜 40%。

输入 $1.50/百万 token，输出 $9.00/百万 token，上下文窗口 100 万 token。而且谷歌自己说，头部企业迁移 80% 负载到 3.5 Flash，年省超 10 亿美元。

这不是一个简单的价格战。这是一个关于「AI 边际成本」的系统性变化。

回想两年前，很多独立开发者在做 AI 产品的时候，最大的焦虑之一是：API 调用费用太贵，用户一多，成本就失控。这个焦虑，正在被大厂的竞争悄悄抹平。

但有一个副作用你要注意：价格降了，竞争对手进场的门槛也降了。

Antigravity 2.0 现场演示里有个细节值得反复咀嚼：93 个 subagent 并行工作，但整个过程的关键不是「agent 能写代码」，而是任务分解的方式——谁来决定怎么拆，拆成哪些子任务，每个子任务要输出什么接口给下一个 agent。

这个角色，他们给起了个名字：Orchestrator（编排者）。

这是独立开发者在 Agent 时代的真实位置：不是执行者，是编排者。你不需要比 Agent 更能写代码，但你需要比 Agent 更懂这个具体业务领域里的问题应该怎么被拆解。

03 / 反直觉：大模型公司花 3 亿买工具公司，是在帮你

Anthropic 收购 Stainless 的新闻，在国内 AI 圈里没激起太大浪花。大家的注意力都在 Google I/O 和 Gemini 3.5 Flash 上。

但我觉得这件事对独立开发者的影响，比 Gemini 快 4 倍更深远。

Stainless 的创始人 Alex Rattray 是前 Stripe 工程师。他深知 API 生态里最麻烦的事不是 API 本身，而是开发者接入的摩擦：文档不够、SDK 没人维护、语言绑定有 bug……每一个摩擦点，都是一个用户流失的理由。

Anthropic 把 Stainless 买进来，等于宣布：Claude 的 SDK 质量，从此是战略优先级，不是边缘维护任务。

对独立开发者来说，这意味着：Claude API 的接入成本会降低，MCP server 的生态会更完善，你搭的那个「把 Claude 接进 Notion/Linear/你自己的产品」的工具，底层基础设施会越来越稳。

更大的反直觉是这个：大厂在争夺基础设施层，恰恰是因为应用层太分散、太难控制。而应用层，正是独立开发者活着的地方。

"大厂的军备竞赛，是独立开发者最便宜的基础设施扩张。"

04 / 这几天 builders 圈值得关注的事

🔧 Antigravity 2.0 正式全球开放

谷歌 I/O 2026 现场直接宣布开放。有三个层次：桌面 App、CLI、SDK。对独立开发者最实用的可能是 CLI + SDK 组合——理解成「Google 版 Claude Code」，配合 Gemini 3.5 Flash 跑起来速度快很多，成本也明显更低。这周值得找时间试一下 Antigravity CLI，特别是如果你有大量代码重构或自动化测试的需求。

🤖 Anthropic 控制了 OpenAI 的 SDK 工厂

Stainless 同时为 OpenAI 和 Google 生成官方 SDK——而现在它归 Anthropic 了。这件事标志着：AI 行业的「中立基础设施」时代正在结束，每一个 SDK、每一个 MCP server，都开始有了立场。关注这件事的后续进展，可以帮你在工具选型时做出更有依据的判断。

⚖️ 马斯克诉 OpenAI 案陪审团裁定败诉

5 月 18 日，加州法院裁定马斯克相关主张超过诉讼时效，OpenAI 免责。这件事本身对独立开发者没有直接影响，但它传递了一个信号：OpenAI 内部不稳定因素减少，接下来他们会把更多精力放回产品上。如果你有基于 OpenAI API 搭的产品，可以期待接下来的更新会更密集。

05 / 如果你今天只做一件事

① 测试 Gemini 3.5 Flash 替换你现有的 AI 调用

不需要重写产品逻辑，只需要把 API endpoint 换一下，跑几个你自己的 benchmark。谷歌说普通调用有 4 倍速度提升，价格便宜 40%。对于成本敏感的独立开发者，这个替换收益是直接可见的。

② 认真研究一下 MCP 协议

Anthropic 用 3 亿美元强调了一件事：MCP 是 AI Agent 时代的「接口标准」。现在学 MCP，就像 2010 年学 REST API——不是可选项，是基础设施认知。谷歌这次也明确说 Gemini Spark 后续会开放第三方 MCP 接入。先搞懂 MCP 的 server 怎么写，你就比 90% 的人多了一个接入点。

③ 给你的产品写一个「Agent 接入文档」

不是给用户看的，是给你自己看的：你的产品，哪些功能可以被 Agent 调用？哪些 API 你还没暴露出来？哪些流程，如果有 Agent 接进来，会省掉最多的手工操作？这个文档写完，你会清晰地看到你的产品在 Agent 时代的「接口密度」。

最后一个意外的利好：

谷歌宣布 Gemini Spark（个人 AI Agent）下周只对美国 Ultra 订阅用户开放 Beta，$100/月。这意味着真正好用的个人 Agent，还没有飞入寻常百姓家。

你不需要比谷歌的 93 个 Agent 更厉害，你只需要比那些还不知道 Agent 存在的用户先走一步。
