"不是降价促销，是永久锁定原价的四分之一。"

这是 DeepSeek 5 月 22 日的公告。原本 5 月 31 日到期的 2.5 折优惠，直接变成永久定价——输入 0.435 美元/百万 token，输出 0.87 美元。对比 Claude Opus 4.7，便宜了差不多 8 到 10 倍。

听起来是个好消息，对吧？

但同一天，另一件事也在发酵。Anthropic 在 5 月 14 日宣布：6 月 15 日起，Claude Code 和 Agent SDK 的程序化用量将从原订阅配额里拆出来，单独立一个账单。Pro 用户每月多给 $20 额度，Max 5x 给 $100，Max 20x 给 $200——超出部分按完整 API 费率计费。

一边是史上最便宜的前沿模型，一边是你每天在用的工具悄悄涨价。

这就是今天这篇文章要聊的矛盾：模型越来越便宜，但你的 AI 账单可能越来越贵。问题不在模型本身——在你有没有一套"路由策略"。

01 / 价格战的终局：模型不再是奢侈品

先看几个数字，感受一下 2026 年 5 月发生了什么。

事件 日期 核心影响
DeepSeek V4-Pro 永久降价 75% 5/22 输入 $0.435，输出 $0.87，缓存命中仅 $0.0036
Cursor Composer 2.5 发布 5/18 SWE-Bench 追平 Opus 4.7，标准版价格仅 1/10
Anthropic 账单拆分 5/14 6/15 起 Agent 用量独立配额，重度用户面临涨价
Gemini 3.5 Flash 发布 5/19 Google I/O 推出，Agent 场景比肩前代 Pro

这四件事放在一起看，信号非常清晰：模型层正在加速"去溢价化"。

Cursor 用 Kimi K2.5 做底座，靠后训练就把 Composer 2.5 拉到 Opus 4.7 的水平，定价却只有十分之一。这证明了什么？证明了一个让 Anthropic 和 OpenAI 不太舒服的事实：编程场景下，模型能力已经接近天花板，边际提升越来越贵，但替代品越来越便宜。

对独立开发者意味着什么？意味着你不再需要一个"全能模型"。你可以把工作拆开：

  • 常规 CRUD、重构、补测试 → DeepSeek V4-Pro，成本几乎可以忽略
  • 复杂架构设计、安全审计 → Opus 4.7 或 Composer 2.5
  • 快速原型、Agent 编排 → Gemini 3.5 Flash

过去我们纠结"用哪个模型最好"，现在要问的是"这个任务值多少钱的模型"。

02 / Anthropic 拆账单，不是涨价——是逼你做选择题

很多人看到 Anthropic 新计费规则的第一反应是"又要多花钱了"。但我认为，这件事的底层逻辑不是涨价，是Anthropic 在逼用户正视一个事实：你的 AI 用量已经大到不能靠订阅价兜底了。

想想看。一个重度 Claude Code 用户，一天跑几十个 Agent 任务，一个月轻松烧掉几百万 token。如果这还用 $20/月的 Pro 套餐兜底，Anthropic 是在亏钱服务。所以拆账单的逻辑很简单：聊天可以补贴，Agent 不行。

这对你的实际影响是什么？

6 月 15 日之前，你需要做一个"用量审计"。 打开你的 Claude 设置，看过去 30 天 Agent SDK 和 claude -p 的用量。算一下在拆分后，多少工作负载会被踢到超额计费区。

一个实操建议：把重复性高、不需要顶级推理能力的 Agent 任务迁移到 DeepSeek V4-Pro。 比如自动生成 commit message、跑 lint 修复、写单元测试——这些场景下，DeepSeek 的输出质量和 Opus 的差距，远小于 10 倍的价格差距。

你省下来的 Opus 额度，留给那些真正需要"一步到位"的任务：架构设计讨论、复杂 bug 排查、安全相关的代码审查。这就是"路由策略"的第一层：按任务关键性分配模型预算。

03 / 最便宜的东西，可能让你付出最大的代价

到这里，你可能觉得结论很简单——"那就多用 DeepSeek，少用 Claude"。但我想给出一个反直觉的提醒。

便宜模型的隐性成本不在 token 价格，在"返工率"。

一个真实场景：你用 DeepSeek V4-Pro 生成了一段认证逻辑。代码能跑，测试通过。但两周后你发现它在 edge case 下有一个 subtle 的安全漏洞——因为模型没有足够深地理解你的整体架构。修复这个漏洞，你花了一个下午。

如果你当初用 Opus 4.7 生成同样的逻辑，可能多花 $0.50 的 token 费，但省下了一个下午。

我不是在说"贵的一定好"。我说的是：路由策略的第二层，是要建立"返工成本"的直觉。

  • 一次性、低风险的代码（脚本、配置、简单 UI 组件） → 便宜模型
  • 会被反复调用的核心逻辑、涉及安全、涉及数据一致性 → 贵模型
  • 不确定的时候 → 先用便宜模型跑一遍，再让贵模型 review 一遍

这个规则不是什么高深的工程实践，它就是独立开发者省时省钱的核心竞争力。

04 / 这几天 builders 圈值得关注的事

Cursor Composer 2.5 —— 不是替代 Claude，是开辟第三条路

Composer 2.5 用 Kimi K2.5 底座 + 后训练，在 SWE-Bench Multilingual 上拿了 79.8%，跟 Opus 4.7 和 GPT-5.5 平起平坐。但标准版价格是 Opus 的十分之一。这件事最有意思的不是"Cursor 追上来了"——是它证明了编程 Agent 的能力越来越不绑定在底层模型上。后训练、提示工程、工具链集成，这些"模型外面的东西"正在成为新的能力杠杆。对独立开发者来说，这意味着：不要把宝押在一个模型上。

Grok Build CLI —— xAI 的编程 Agent 入场了

5 月 14 日 xAI 发布了 Grok Build CLI，正面挑战 Claude Code。亮点：2M token 上下文窗口、支持 8 个并行子 Agent、Headless 模式。但目前仅限 SuperGrok Heavy 订阅（$299/月），前六个月优惠 $99/月。门槛不低，但如果你的工作流重度依赖 Agent 编排，这个"子 Agent 并行"的能力值得关注。它代表了一个趋势：编程 Agent 正在从"单线程助手"变成"多线程工头"。

Qwen 3.7-Max —— 中文场景的新选择

阿里在 5 月 20 日发布了 Qwen 3.7-Max 预览版，1M 上下文，LM Arena 总榜 #13，数学 #7。在 Artificial Analysis 智能指数上拿到 56.6，是中文模型里的最高分。如果你在开发中文内容产品、中文客服 Agent、或者需要处理大量中文文档的场景，这个模型值得一试。OpenRouter 上定价 $2.50/$7.50，性价比不错。

05 / 如果你今天只做一件事

第一件事：做一次"AI 用量审计"。 打开你所有 AI 工具的账单页面（Claude、Cursor、OpenAI API），把过去 30 天的用量拉出来。分三列标注：哪些是"高频但低风险"的调用、哪些是"低频但关键"的调用、哪些是"可以砍掉"的冗余调用。然后做一件事：把第一列的默认模型切到 DeepSeek V4-Pro。

第二件事：设置一个成本锚点。 给自己定一个每月的 AI 工具总预算。比如 $50/月。Claude Pro $20 + Cursor $20 + API 额度 $10。然后每个月初看一下，哪个工具快超了，主动做负载迁移。不要让账单在月底"惊喜"你。

第三件事：盯着 6 月 15 日。 如果 Anthropic 的账单拆分生效后，你的月成本涨幅超过 30%，那就说明你的 Agent 用量已经大到需要一套正式的"模型路由策略"了。别等到账单来了再慌。

最后，送一个意外的利好。

价格战越凶，独立开发者的优势越大。 因为大公司的开发团队有固定的技术选型流程和审批链，切换一个模型可能要开三次会。而你只需要改一行配置。

当 DeepSeek 把价格砸到四分之一，Cursor 把能力拉到 Opus 级别，Anthropic 的账单逼你精打细算——所有这些变化，对"单兵作战"的人来说，都是武器库的扩充，而不是威胁。

你的竞争对手不是 Anthropic 或 DeepSeek。是那些还不知道怎么路由模型的其他独立开发者。
