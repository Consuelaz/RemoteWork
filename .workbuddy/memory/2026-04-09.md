## 下午修复：海外岗位显示 0（13:48）

### 根本原因
1. **早晨截断脚本（`truncate_desc.py`）破坏 JS 语法**：用正则 `re.sub(r"description: '((?:[^'\\]|\\.)*)'", ...)` 截断时，正则匹配到 description 内的 `\'`（转义单引号）就提前停止，截断位置不在字段边界，导致单引号不闭合，JS 语法错误，`JOBS_GLOBAL` 变量无法定义 → 海外岗位显示 0
2. **`load_js_array` 缺少 unescape**：读取历史 JS 文件时，字段值保留了 JS 转义状态（如 `\\n`），写回时 `escape_str()` 再次转义，每次运行多一层 `\\` → 历史数据积累后 description 变成乱码

### 修复方案
- `scrape.sh` 第1281行：读取历史数据后立即 unescape（还原 `\\n` → `\n`、`\\'` → `'` 等），让数据以原始文本形式流转
